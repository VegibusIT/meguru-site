MEGURU API Reference

共同配送インフラ MEGURU の REST API ドキュメントです。

概要

MEGURUは荷主からの配送依頼を受け、Dispatch Engineが自動で最適経路を計算し、配送レグを生成します。荷主は出発地・到着地・日付・ケース数だけ指定すれば、ルートの選択は不要です。

認証

運営者・ドライバー（JWT）

// ログインしてトークンを取得
POST /auth/login
{
  "tenant_id": "uuid",
  "role": "admin"
}

// 以降のリクエストにヘッダーを付与
Authorization: Bearer eyJhbGci...

荷主（APIキー）

// テナント管理者が発行したAPIキーをヘッダーに付与
X-API-Key: mk_live_xxxxxxxxxxxx

Base URL

https://meguru-api.fly.dev

テナント作成

POST /admin/tenants

新しいエリア運営者を登録します。

// Request
{
  "name": "やさいバス静岡",
  "plan": "standard"
}

// Response: 201 Created
{
  "id": "202c55e5-8cce-468d-866c-bba8aa8f788b",
  "name": "やさいバス静岡",
  "plan": "standard",
  "active": true,
  "created_at": "2026-03-23T04:12:49Z"
}
FieldTypeDescription
namestring必須テナント名
planstring必須starter / standard / pro / enterprise

テナント一覧

GET /admin/tenants

バス停作成

POST /stops?tenant_id={uuid}

バス停（集荷・配達・積替ポイント）を登録します。

{
  "name": "長沼町農場",
  "address": "北海道長沼町",
  "location": { "latitude": 43.0, "longitude": 141.7 },
  "stop_type": "collection",
  "capacity_cases": 100
}
stop_type説明
collection集荷専用（農家など）
delivery配達専用（飲食店など）
transit積替拠点（中継バス停）
both集荷＋配達両用
garage車庫（ルート起終点）

バス停一覧

GET /stops?tenant_id={uuid}

ルート作成

POST /routes?tenant_id={uuid}

便（ルート）を登録します。運行曜日・時刻・温度帯・積載上限を設定。

{
  "name": "ルートA（農場→マオイ）",
  "schedule": {
    "days_of_week": ["monday", "wednesday", "friday"],
    "departure_time": "08:00:00"
  },
  "stops": [
    { "stop_id": "uuid-farm", "sequence": 1 },
    { "stop_id": "uuid-transit", "sequence": 2 }
  ],
  "temperature": "ambient",
  "capacity_cases": 50
}

ルート一覧

GET /routes?tenant_id={uuid}

バス停接続の定義

POST /connections?tenant_id={uuid}

バス停間の接続を定義します。Dispatch Engineはこのグラフから経路を探索します。

{
  "from_stop_id": "uuid-farm",
  "to_stop_id": "uuid-transit",
  "days_of_week": ["monday", "wednesday", "friday"],
  "transit_days": 0,          // 0=当日, 1=翌日
  "active_from": "2026-01-01",
  "active_until": null         // null=無期限
}

一括インポート

POST /connections/bulk?tenant_id={uuid}

複数の接続を一括登録します。

{
  "connections": [
    { "from_stop_id": "...", "to_stop_id": "...", ... },
    { "from_stop_id": "...", "to_stop_id": "...", ... }
  ]
}

配送依頼（最重要API）

POST /shipments?tenant_id={uuid}

荷主が配送を依頼します。ルート指定は不要。Dispatch Engineが自動で最適経路を計算し、ShipmentLegを生成します。

// Request — 出発地・到着地・日付・ケース数だけ
{
  "external_order_id": "YB-12345",    // 任意: 自社の注文ID
  "origin_stop_id": "693ec6db-...",
  "destination_stop_id": "25c2304e-...",
  "scheduled_date": "2026-04-06",
  "container": { "cases": 3, "size": "medium" },
  "temperature": "ambient"
}

// Response: 201 Created
// Dispatch Engineが自動で2レグに分割（積替経由・翌日配送）
{
  "id": "784c4853-...",
  "status": "confirmed",
  "legs": [
    {
      "leg_order": 1,
      "route_id": "df7b5a7e-...",
      "from_stop_id": "693ec6db-...",  // 長沼町農場
      "to_stop_id": "19e2dfa7-...",    // 道の駅マオイ（積替）
      "scheduled_date": "2026-04-06",   // 月曜
      "status": "pending"
    },
    {
      "leg_order": 2,
      "route_id": "387ab540-...",
      "from_stop_id": "19e2dfa7-...",  // 道の駅マオイ
      "to_stop_id": "25c2304e-...",    // 大丸札幌店
      "scheduled_date": "2026-04-07",   // 火曜（翌日）
      "status": "pending"
    }
  ]
}

配送追跡

GET /shipments/{id}

配送の現在のステータスとレグ情報を取得します。

Status説明
pending受付済み（経路未確定）
confirmed経路確定・便アサイン済み
picked_up集荷済み
in_transit輸送中
delivered配達完了
cancelledキャンセル済み
failed配送失敗

キャンセル

PATCH /shipments/{id}/cancel

配送をキャンセルします。集荷済み以降はキャンセル不可。

// カスケード処理:
// 1. Shipment.status → cancelled
// 2. 全ShipmentLeg.status → cancelled
// 3. 積載枠を解放
